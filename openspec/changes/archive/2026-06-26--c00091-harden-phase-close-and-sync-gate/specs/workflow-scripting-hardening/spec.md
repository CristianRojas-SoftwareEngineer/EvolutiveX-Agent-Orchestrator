## Non-canonical record

Tres adiciones de infraestructura de scripting al workflow specification-delta. Ninguna tiene un requisito en `openspec/specs/`; son mejoras de robustez a las herramientas internas del pipeline.

- `sync-completion-gate` — Nuevo gate determinista post-`synchronize` (etapa 9). El closer actualmente invoca `synchronize` → `archive` sin ninguna validación entre ambos. Se añade validación: extender `ARTIFACT_ORDER` en `scripting/openspec/verify-stage-completion.ts` con el nivel `'synchronized'` (que comprueba que los archivos de `openspec/specs/<cap>/spec.md` afectados existen y están actualizados, y que los docs en `openspec/` han sido escritos por synchronize) **o** crear un script independiente `scripting/openspec/verify-sync-completion.ts` invocado explícitamente por el closer antes de archive. La elección entre ambas estrategias se resuelve en `design.md`. El gate es no-funcional para el producto final; aplica solo durante la ejecución del pipeline de planificación.

- `close-phase-script` — Nuevo script compartido `scripting/openspec/close-phase.ts` que encapsula los tres efectos de cierre de fase que hoy cada subagente implementa como prosa improvisada con bloques `node -e` inline: (1) `writePhaseMarker(phase, change)` via `read-phase-marker.ts`, (2) escritura atómica del sidecar `openspec/.workbench/<phase>.timings.json` (tmp + rename), y (3) para el closer exclusivamente, limpieza de workbench. **Defecto observado en producción:** el planner escribió duraciones sintéticas redondas (10s/20s/30s/60s/180s, total exacto 300 000 ms) en vez de las duraciones reales (~9m 25s, 565 294 ms reales); el explorer escribió `startedAt == completedAt` y `durationMs: 0` pese a que la fase tomó ~2m 55s reales. Esto ocurre porque cada subagente inventa los valores inline en vez de recibirlos de una fuente autoritativa. La fuente autoritativa es el orquestador (campo `duration_ms` del `tool_result.usage` del Agent tool). `close-phase.ts` DEBE recibir la duración real como parámetro (`--duration-ms <n>` o `--started-at <iso>`) y rechazar valores no finitos, negativos o iguales a cero cuando la fase tuvo duración real no nula. El script es la única ruta de escritura del sidecar; los subagentes nunca calculan ni inventan duraciones. Las cuatro definiciones de subagente (explorer, planner, implementer, closer) migran a invocar este script con un único comando bash sin bloques `node -e`.

- `explorer-timings-type-fix` — En `.claude/agents/explorer-specification-delta.md`, el placeholder `durationMs: '<%= it.harnessDurationMs %>'` está entre comillas simples, produciendo un valor de tipo `string` en el JSON generado. El schema de timings en `openspec/specs/orchestrator-stage-timings/spec.md` exige que `durationMs` sea un número finito. Este fix queda subsumido en la migración a `close-phase.ts` (el script recibe el valor real del harness como número, nunca como string); si por cualquier razón la migración no cubre el explorer, se aplica como corrección aislada eliminando las comillas del placeholder en la línea 168 del agente del explorer.
